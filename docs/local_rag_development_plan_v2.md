# [리걸 AI 서비스] 로컬 RAG 시스템 개발 계획서 (v2)

본 계획서는 완전 오프라인 및 비용 제로를 지향하는 **로컬 법률 질의응답 RAG 시스템 v2**의 설계서입니다. 1차 정규식 전방 탐색과 2차 문장 임베딩 유사도 비교를 결합한 **하이브리드 세만틱 청킹(Hybrid Semantic Chunking)** 모델을 사용하고, 데이터 베이스 저장소를 물리적으로 분리 구축하는 사양을 명세합니다.

---

* **적용 데이터:** `01.원천데이터` 내 `TS_결정례`, `TS_법령`, `TS_판결문`, `TS_해석례` CSV 파일.

## 2. 하이브리드 세만틱 청킹 아키텍처 규칙 (Hybrid Chunking Rule)

단순한 글자 수 기반의 청킹이나 구분자 소실 위험이 있는 문자열 split 방식을 배제하고, 수학적 유사성과 리걸 고유의 계층 구조를 보존하는 2단계 청킹 전략을 가동합니다.

```
[원천 텍스트 본문]
       │
       ▼ (1단계: 정규식 전방 탐색 lookahead 분할)
[거대 의미 블록 분리 (제X조, 【 등 구분자 보존)]
       │
       ├─► [1,500자 이하 블록] ──► (그대로 보존 통과) ──┐
       │                                            │
       └─► [1,500자 초과 블록] ──► (2단계 분할 적용)      │
                                       │            │
             ┌─────────────────────────┘            │
             ▼ (문장 단위 분리 및 bge-m3 임베딩)         │
       [문장 간 코사인 유사도 연산]                     │
             │                                      │
             ▼ (유사도 < 0.80 지점에서 단락 분할)        │
       [의미 세분화 청크 완성] ───────────────────────┴──► [최종 Chroma DB v2 저장]
```

### ① 1차 방어: 정규표현식 전방 탐색 분할
* 법적 조문의 뼈대인 글자(예: '제', '【')가 파손되는 것을 방지하기 위해 긍정 전방 탐색 `(?=...)` 정규식 분할을 사용합니다.
* **법령**: `r'\n(?=제\d+조)'` 조항 직전 위치
* **판결문**: `r'\n(?=【)'` 판시사항/판결요지 직전 위치
* **해석례**: `r'\n(?=질의요지|회답|이유)'` 본문 직전 위치

### ② 2차 최적화: 임베딩 유사도 기반 단락 분 분해
* 1차 분할된 블록 중 **1,500자를 초과하는 거대 텍스트 블록**에 한해서만 의미 전환 기준 세분화 분할을 적용합니다.
* 블록 내부의 문장을 `bge-m3` 임베딩을 이용해 실수 벡터화하고, 이웃한 문장 간의 **코사인 유사도(Cosine Similarity)**를 구합니다.
* 유사도가 설정된 기준선(**0.80**) 미만으로 떨어지는 구간을 탐색하여, 해당 인접 영역을 분할 경계선으로 결정합니다. 단, 너무 짧은 파편화를 막기 위해 최소 500자 크기 한도를 적용합니다.
